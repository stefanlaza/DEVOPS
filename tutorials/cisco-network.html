Overview

Three-Tier Campus Design

Enterprise campus networks follow a three-tier hierarchy that separates switching functions into distinct layers. Each tier has a specific role and connects only to the tiers directly above and below it.

Edge / Firewall

Redundant ASA 5525-X firewalls in active/standby. Internet edge, DMZ, VPN termination.

Core Layer

Catalyst 9500 switches in VSS. High-speed Layer 3 forwarding between distribution blocks.

Distribution Layer

Catalyst 9300 switches. VLAN routing, ACL enforcement, QoS policy, HSRP gateway redundancy.

Access Layer

Catalyst 9200 switches. Edge port security, 802.1X authentication, PoE for phones and APs.

VLAN Design

VLAN Segmentation Plan

VLANs segment broadcast domains and enforce security boundaries between different types of traffic. Every device is assigned to a VLAN based on its function — never its physical location.

Security Note

Always change the native VLAN from VLAN 1 to an unused VLAN (99 here) on all trunk ports. VLAN 1 is the default and a common target for VLAN hopping attacks.

Configuration Guide

01

ASA Firewall — Active/Standby Failover

Configure two ASA 5525-X firewalls in active/standby mode. The standby unit mirrors the active config and takes over automatically if the primary fails.

! Primary ASA — interface configuration
interface GigabitEthernet0/0
 nameif outside
 security-level 0
 ip address 203.0.113.1 255.255.255.0 standby 203.0.113.2

interface GigabitEthernet0/1
 nameif inside
 security-level 100
 ip address 10.0.0.1 255.255.255.0 standby 10.0.0.2

interface GigabitEthernet0/2
 nameif dmz
 security-level 50
 ip address 192.168.1.1 255.255.255.0 standby 192.168.1.2

! Failover configuration
failover
failover lan unit primary
failover lan interface FAILOVER GigabitEthernet0/3
failover link FAILOVER GigabitEthernet0/3
failover interface ip FAILOVER 10.99.0.1 255.255.255.252 standby 10.99.0.2
02

Core Switch — VSS Configuration

Configure the two Catalyst 9500 switches as a Virtual Switching System (VSS). VSS presents two physical switches as a single logical switch, eliminating STP blocking and doubling bandwidth.

! Switch 1 — Core-SW1
switch virtual domain 100
 switch 1
 switch 1 priority 110

interface TenGigabitEthernet1/0/1
 description VSL-to-Core-SW2
 switch virtual link 1
 no shutdown

! Enable VSS
switch convert mode virtual

! Core-SW1 uplinks to distribution (port-channel)
interface port-channel10
 description To-DIST-SW1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk native vlan 99
 switchport trunk allowed vlan 10,20,30,40,50,60,70
03

Distribution Layer — Inter-VLAN Routing and HSRP

Distribution switches perform Layer 3 routing between VLANs. Configure HSRP so each VLAN has a virtual gateway IP that fails over between the two distribution switches.

! Distribution SW1 — VLAN SVIs with HSRP
interface Vlan30
 description USERS-Gateway
 ip address 10.30.0.2 255.255.248.0
 standby version 2
 standby 30 ip 10.30.0.1        ! Virtual gateway — this is what clients use
 standby 30 priority 110        ! Higher = preferred
 standby 30 preempt
 standby 30 authentication md5 key-string YOUR-KEY
 ip helper-address 10.20.0.10   ! DHCP server

interface Vlan40
 description VOICE-Gateway
 ip address 10.40.0.2 255.255.252.0
 standby version 2
 standby 40 ip 10.40.0.1
 standby 40 priority 110
 standby 40 preempt
 ! QoS — trust DSCP from IP phones
 mls qos trust dscp
04

Access Layer — Port Security and 802.1X

Access switches connect end devices. Configure port security to limit MAC addresses per port, and use 802.1X to authenticate users before granting network access.

! Access switch — user port configuration
interface GigabitEthernet1/0/1
 description USER-WORKSTATION
 switchport mode access
 switchport access vlan 30
 switchport nonegotiate          ! Disable DTP — prevent VLAN hopping
 spanning-tree portfast          ! Skip STP listening/learning for end devices
 spanning-tree bpduguard enable  ! Shut port if it receives a BPDU (rogue switch)
 ip dhcp snooping limit rate 15  ! Limit DHCP requests per second
 storm-control broadcast level 20.00  ! Block broadcasts over 20%
 no shutdown

! IP Phone port — voice + data VLAN
interface GigabitEthernet1/0/5
 description IP-PHONE-with-PC
 switchport mode access
 switchport access vlan 30       ! Data VLAN for PC behind phone
 switchport voice vlan 40        ! Voice VLAN for phone itself
 spanning-tree portfast
 spanning-tree bpduguard enable
 power inline auto               ! Enable PoE
05

OSPF Routing

Configure OSPF between the core, distribution, and firewall to dynamically exchange routing information. Use area 0 (backbone) for the core and separate areas for each distribution block.

! Distribution SW1 — OSPF configuration
router ospf 1
 router-id 10.10.10.2
 auto-cost reference-bandwidth 10000  ! Set for 10Gbps links

 ! Advertise all VLAN subnets
 network 10.20.0.0 0.0.3.255 area 0
 network 10.30.0.0 0.0.7.255 area 0
 network 10.40.0.0 0.0.3.255 area 0

 ! Suppress OSPF hellos on access ports (not needed)
 passive-interface default
 no passive-interface GigabitEthernet1/0/1  ! Uplink to core

! Authenticate OSPF neighbors (prevent route injection)
interface GigabitEthernet1/0/1
 ip ospf authentication message-digest
 ip ospf message-digest-key 1 md5 YOUR-OSPF-KEY
06

Trunk Configuration Between Layers

All inter-switch links carry multiple VLANs using 802.1Q trunking. Explicitly allow only the VLANs needed on each trunk — never use switchport trunk allowed vlan all.

! Distribution to Core trunk
interface GigabitEthernet1/0/48
 description UPLINK-TO-CORE
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport nonegotiate
 switchport trunk native vlan 99
 switchport trunk allowed vlan 10,20,30,40,50,60,70
 channel-group 10 mode active  ! LACP port-channel

! Access to Distribution trunk
interface GigabitEthernet1/0/47
 description UPLINK-TO-DIST
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport nonegotiate
 switchport trunk native vlan 99
 switchport trunk allowed vlan 10,30,40  ! Only VLANs this access block needs
Common Mistake

Forgetting switchport nonegotiate on trunk ports leaves DTP (Dynamic Trunking Protocol) active. DTP can be exploited for VLAN hopping. Always disable it explicitly.
